# 市场调研：AI 学习资源小工具

- 生成日期：2026-06-28
- 定位：AI 学习资源运营台
- 目标用户：Windows 本地模型 / Vibe Coding 学习者

## 未明显饱和的切入赛道

| 赛道 | 饱和度 | 为什么还有机会 | 与当前项目匹配度 |
| --- | --- | --- | --- |
| Windows 本地模型新手上手 | 中低 | 多数内容停留在工具介绍，缺少面向 Windows 用户的排错、模型选择和工作流闭环。 | 与现有 Ollama、Qwen、DeepSeek、部署资源高度匹配。 |
| Vibe Coding 学习路径与模板 | 中低 | 泛提示词库很多，但“从需求到页面/脚本/部署”的可复制训练路径仍不足。 | 可用提示词模板、代码案例、自动化脚本案例形成系列内容。 |
| AI 自动化脚本案例库 | 低 | 真实小任务脚本分散在博客和仓库里，缺少按难度、场景、可复制命令组织的中文入口。 | 适合做长尾 SEO 与资源订阅。 |
| 本地 AI 工具运营导航 | 中 | 工具导航站拥挤，但面向“本地部署 + 学习 + 自动运营”的组合还不常见。 | 能与本项目自动更新、资源质检、日报生成形成差异。 |

## 推荐主线

优先选择“Windows 本地模型新手上手 + Vibe Coding 学习路径 + 自动化脚本案例库”的组合，不做泛泛 AI 工具导航。这个组合更容易形成可搜索、可复用、可持续更新的内容资产。

## MVP 内容策略

1. 每周新增 3 条可复制资源，必须包含命令、提示词或操作步骤。
2. 每条资源至少归入一个学习路径：本地模型、代码模型、提示词、自动化、部署。
3. 每次更新生成运营简报，记录新增资源、缺口和下一步内容选题。
4. 广告和分销先预留位置，等有稳定访问后再接入。
