# 手动安装与部署准备

因为这次自动安装步骤被外部审批系统拦住了，所以我改成给你一份能直接照着做的手动流程。你按这个顺序补齐就行。

## 一、先安装 Git

### 方式 1：官网安装

搜索官方安装包关键词：
- `Git for Windows`

安装完成后，在 PowerShell 里验证：

```powershell
git --version
```

### 方式 2：你自己手动执行 winget

```powershell
winget install --id Git.Git -e
```

### Git 装完后要达到的状态

- `git --version` 能显示版本号
- 之后可以把网页代码推到 GitHub

## 二、再安装 Ollama

### 方式 1：官网安装

搜索官方安装包关键词：
- `Ollama Windows download`

安装完成后，在 PowerShell 里验证：

```powershell
ollama --version
```

### 方式 2：你自己手动执行 winget

```powershell
winget install --id Ollama.Ollama -e
```

### Ollama 装完后的推荐动作

先拉一个轻量模型测试：

```powershell
ollama pull qwen2.5:7b
```

如果你后面主要拿来写代码，再补一个代码模型：

```powershell
ollama pull qwen2.5-coder:7b
```

### Ollama 装完后要达到的状态

- `ollama --version` 能显示版本号
- `ollama list` 可以正常返回模型列表
- 至少本地有一个能跑的模型

## 三、GitHub 账号准备

安装 Git 后，你就可以开始做代码托管了。

建议：
- 先注册 `GitHub`
- 新建一个公开仓库
- 仓库名称可以先用：`ai-learning-resource-tool`

后面我可以继续帮你：
- 初始化 Git 仓库
- 生成首版提交
- 整理适合 Cloudflare Pages 的目录结构

## 四、Cloudflare Pages 准备

你需要一个 `Cloudflare` 账号，后面把 GitHub 仓库接进去即可。

你要达到的状态：
- 能登录 Cloudflare
- 能进入 `Pages`
- 能选择 GitHub 仓库做部署

## 五、当前机器的命令注意事项

### npm 在 PowerShell 里的使用方式

这台机器直接执行 `npm` 会被脚本策略拦住，所以请优先使用：

```powershell
npm.cmd -v
npm.cmd run check:env
npm.cmd run preview
```

### PowerShell 脚本执行方式

本项目已经给你准备了脚本，运行方式如下：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\check-env.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\start-preview.ps1
```

## 六、建议你安装完后马上做的三件事

1. 执行环境检查脚本，确认 `Git` 和 `Ollama` 都显示已就绪
2. 本地启动预览，确认网页骨架能正常打开
3. 把账号注册清单里前两项先做掉：`GitHub`、`Cloudflare`

## 七、如果你想走最低上线版

你只需要先完成这些：
- Git
- Ollama
- GitHub
- Cloudflare

这样就够开始做一个公开可访问的小站了。
